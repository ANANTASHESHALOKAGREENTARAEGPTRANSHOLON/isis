The `@Collection` annotation applies to collections collecting together all domain semantics within a single annotation.

The table below summarizes the annotation's attributes.

.`@Collection` attributes
[cols="2,2,4a", options="header"]
|===

| Attribute
| Values (default)
| Description


|xref:_ug_reference-annotations_manpage-Collection_domainEvent[`domainEvent()`]
|subtype of `CollectionDomainEvent` +
(`CollectionDomainEvent.Default`)
|the event type to be posted to the xref:_ug_reference-services-api_manpage-EventBusService[`EventBusService`] to broadcast the collection's business rule checking (hide, disable, validate) and its modification (before and after).


|xref:_ug_reference-annotations_manpage-Collection_editing[`editing()`]
|`ENABLED`,`DISABLED`,`AS_CONFIGURED` +
(`AS_CONFIGURED`)
|whether a collection can be added to or removed from within the UI


|`editingDisabledReason()`
|String value
|if `editing()` is `DISABLED`, provides a reason as to why.
